## 3. 核心逻辑测试

### 3.1 千人千卷测试

**测试目标：** 验证不同学生获得不同题目顺序

**测试步骤：**
1. 教师创建试卷，配置组卷规则
2. 发布试卷
3. 学生A开始考试，记录题目顺序
4. 学生B开始考试，记录题目顺序
5. 对比两次题目顺序

**预期结果：**
- 两次题目顺序不同
- 选项顺序不同
- 题目数量相同
- 总分相同

### 3.2 实时倒计时测试

**测试目标：** 验证倒计时准确性

**测试步骤：**
1. 开始考试，记录开始时间
2. 观察倒计时变化
3. 等待1分钟，对比系统时间

**预期结果：**
- 倒计时每秒递减
- 与实际时间误差<1秒
- 时间到自动交卷

### 3.3 草稿防丢档测试

**测试目标：** 验证断网后草稿保存

**测试步骤：**
1. 开始考试，作答3题
2. 断开网络
3. 继续作答2题
4. 恢复网络
5. 刷新页面

**预期结果：**
- 断网时localStorage有草稿
- 恢复网络后自动同步
- 刷新后答案不丢失

### 3.4 AI阅卷测试

**测试目标：** 验证AI自动批改主观题

**前置条件：** 已配置通义千问API Key

**测试步骤：**
1. 创建包含主观题的试卷
2. 学生提交主观题答案
3. 教师触发AI阅卷
4. 等待AI批改完成

**预期结果：**
- AI评分数值合理（不超过满分）
- AI评语内容相关
- 记录到ai_grading_log表
- 教师可覆盖AI评分

## 4. 接口测试（使用Swagger）

### 4.1 访问Swagger

```
http://localhost:8080/swagger-ui.html
```

### 4.2 接口测试流程

1. **登录获取Token**
   - 调用 `POST /api/auth/login`
   - 复制返回的token

2. **设置Token**
   - 点击Swagger右上角 "Authorize"
   - 输入 `Bearer {token}`

3. **测试各接口**
   - 点击接口展开详情
   - 点击 "Try it out"
   - 填写参数
   - 点击 "Execute"

### 4.3 关键接口测试

**测试抽题算法：**
```
POST /api/exam/start/1
```

**测试AI阅卷：**
```
POST /api/grading/papers/1/ai-grade
```

**测试Excel导入：**
```
POST /api/questions/import
选择文件 + subjectId参数
```
